# Security Policy

## Supported Versions

**next.dj** is under active development. Security fixes are applied to the latest release line when practical. We recommend running the latest stable release and keeping Django and other dependencies up to date.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x: Best effort only |

If you need a fix backported to an older line, open an issue so we can discuss feasibility.

## Reporting a Vulnerability

**Please do not** report security issues through public GitHub issues, discussions, or pull request comments. That can put users at risk before a fix exists.

### GitHub Security Advisories

1. Open **[Security advisories](https://github.com/next-dj/next-dj/security/advisories)** for this repository (if you have access).
2. Click **Report a vulnerability** and describe the issue with enough detail to reproduce or reason about impact.

## What to Expect

- We will acknowledge receipt as soon as we can (typically within a few business days).
- We will work on a fix and coordinate disclosure. Please give us reasonable time to patch before public disclosure.
- We may ask follow-up questions or request a retest of a proposed fix.

## Scope

This policy applies to the **next.dj** library and its documented usage with supported Django versions. Third-party applications built with next.dj are out of scope unless the flaw is in next.dj itself.

## Safe Harbor

We appreciate responsible disclosure. If you act in good faith—avoiding privacy violations, destruction of data, or interruption of service—we will not pursue legal action for research that follows this policy.

---

Thank you for helping keep **next.dj** and its users safe.
